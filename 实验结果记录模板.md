# 实验结果记录模板

## 1. 使用说明

这份模板用于统一记录 `UOT_OSDA` 在不同数据集、不同参数设置下的实验结果。

建议每次启动一个新实验时：

1. 先复制一份下面的模板块
2. 填写运行命令与修改参数
3. 训练结束后补齐日志观察、最优结果和结论

这样后续做：

- 参数回溯
- 消融整理
- 论文表格汇总
- 异常排查

都会轻松很多。

## 3. 推荐的实验命名规则

建议统一使用下面的命名格式：

```text
UOT_<source>_<target>_seed<seed>_exp<id>
```

例如：

```text
UOT_Houston13_Houston18_seed2_exp01
UOT_PaviaU_PaviaC_seed7_exp03
```

这样后续日志、截图、表格都方便对应。

## 6. 推荐的首次填表顺序

建议按下面顺序开始填：

1. `PaviaU_7gt -> PaviaC_OS`
2. `Yancheng_ZY -> Yancheng_GF`
3. `Houston13_7gt -> Houston18_OS`
4. `HyRank_source -> HyRank_target`

原因是：

- 先用相对更稳的数据集验证训练链路
- 再逐步进入更难的开放集场景

---

## 7. 额外建议

- 每次实验尽量只改 `1~2` 个参数
- 不要在第一次就同时改：
  - `uot_epsilon`
  - `dustbin_quantile`
  - `threshold_quantile`
  - `uot_loss_weight`
- 如果实验失败，也照样记录，不要只记成功结果
- 记录失败实验，往往比记录成功实验更有价值
